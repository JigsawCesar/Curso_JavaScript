Aula 1

Introdução ao JavaScript e ao Terminal:

    Na Aula 01 foi apresentada a base do ambiente de desenvolvimento em JavaScript, diferenciando a linguagem 
JavaScript do Node.js, que é o ambiente de execução responsável por interpretar e rodar o código fora do 
navegador. Também foram introduzidos os conceitos fundamentais de terminal e sistema de arquivos, 
enfatizando que o computador executa apenas comandos exatos. A aula focou na navegação e manipulação de 
diretórios por meio dos principais comandos do terminal — pwd, ls, cd, cd .., mkdir, touch, rm e rm -r — 
permitindo criar, acessar, listar e remover arquivos e pastas, estabelecendo a base prática para organizar 
projetos em JavaScript.


1. JavaScript:

    JavaScript é uma linguagem de programação utilizada para desenvolver aplicações web, sistemas backend, 
automações e diversas outras soluções. Originalmente criada para rodar no navegador, hoje também pode ser 
executada diretamente no computador por meio de um interpretador.

2. Node.js:

    Node.js é um ambiente de execução (runtime) que permite rodar JavaScript fora do navegador. Ele 
interpreta e executa arquivos JavaScript diretamente no sistema operacional.

Diferença importante: JavaScript é a linguagem.
                      Node.js é o interpretador que executa a linguagem no computador.

3. Conceito de Terminal:

    O terminal é uma interface de linha de comando utilizada para interagir diretamente com o sistema 
operacional. O computador não interpreta intenção, apenas comandos exatos. Qualquer erro de digitação
pode impedir a execução do comando.

Diretório = Pasta

~ representa a pasta raiz do usuário (home)
.. representa o diretório anterior (diretório pai)

Aula 2
Manipulação de Arquivos e Diretórios no Terminal

Resumo:
Nesta etapa da aula foram aprofundados os comandos de manipulação de arquivos e diretórios no terminal, destacando o uso do echo para exibir mensagens e escrever conteúdos em arquivos (utilizando > para sobrescrever e >> para adicionar informações), do cat para visualizar arquivos no terminal, do cp para copiar arquivos entre pastas e do mv para mover, reorganizar e também renomear arquivos e diretórios. Também foram trabalhados os conceitos de caminho absoluto (informando o endereço completo desde a raiz do sistema) e caminho relativo (baseado na posição atual no terminal), além do uso de rmdir para remover pastas vazias, consolidando a compreensão sobre organização e estrutura de projetos no ambiente de desenvolvimento.

1. ECHO
O comando echo é utilizado para exibir mensagens no terminal.
 É muito usado em scripts para informar ao usuário o que está acontecendo durante a execução de um processo (por exemplo, carregamentos ou etapas de instalação).
Exibir mensagem no terminal
echo "Processando arquivo..."

Criar um arquivo com conteúdo
O operador > cria um arquivo e escreve o conteúdo nele.
echo "Olá, mundo" > teste.txt
Explicação:
echo "Olá, mundo" → gera a mensagem


> → redireciona a saída para um arquivo


teste.txt → arquivo criado (ou sobrescrito)


Se o arquivo já existir, ele será substituído.

Adicionar conteúdo sem apagar o anterior
O operador >> adiciona conteúdo ao final do arquivo.
echo "LionsDev" >> teste.txt
Diferença:
> substitui


>> adiciona



2. CAT
O comando cat exibe o conteúdo de um arquivo no terminal.
cat teste.txt
Função:
 Permite visualizar rapidamente o conteúdo de arquivos de texto.

3. CP (Copy)
O comando cp é utilizado para copiar arquivos.
Sintaxe geral:
cp caminho/arquivo.ext pasta-destino
Exemplo:
cp aula1/index.js aula2
Explicação:
Copia o arquivo index.js


Da pasta aula1


Para a pasta aula2



4. MV (Move)
O comando mv é utilizado para mover arquivos ou pastas.
 Também pode ser usado para renomear.

4.1 Mover arquivo para outra pasta
mv pasta/arquivo.ext pasta-destino
Exemplo:
mv aula1/index.js aula2

4.2 Mover e renomear ao mesmo tempo
mv pasta/arquivo.ext pasta-destino/novo-nome.ext
Exemplo:
mv aula1/index.js aula2/app.js

4.3 Renomear arquivo
mv arquivo.ext novo-nome.ext
Exemplo:
mv teste.txt arquivo.txt

4.4 Movimentação usando ponto (.)
O ponto . representa o diretório atual.
mv pasta2/pasta3 .
Significa:
 Mover pasta3 para o diretório atual.

4.5 Movimentação entre subpastas
mv aula2/pasta3/pasta2 aula2
Significa:
 Mover pasta2 (que está dentro de pasta3) diretamente para aula2.

5. Caminho Absoluto e Caminho Relativo
5.1 Caminho Absoluto
É o caminho completo desde a raiz do sistema.
Exemplo:
cd /home/LIONS_CONVIDADO/murillo/Aulas
Características:
Começa da raiz /


Independe do diretório atual



5.2 Caminho Relativo
É baseado no diretório atual.
Exemplo:
cd ../pasta2
Explicação:
.. volta um diretório


Não precisa informar o caminho completo



6. RMDIR
O comando rmdir também pode ser utilizado para remover pastas.
rmdir pasta
Importante:
Só remove pastas vazias.


Se houver arquivos dentro, será necessário usar rm -r.



Conceitos Importantes
echo pode exibir mensagens e escrever em arquivos.


> substitui conteúdo.


>> adiciona conteúdo.


cp copia arquivos.


mv move ou renomeia.


. representa o diretório atual.


Caminho absoluto começa da raiz.


Caminho relativo depende da posição atual.


rmdir remove apenas pastas vazias.

Aula 5
Boolean, Operadores de Comparação e Operadores Lógicos
Resumo:
Nesta aula foram aprofundados os conceitos de tipo booleano e operadores lógicos em JavaScript, explorando comparações entre valores utilizando == (compara apenas valores) e === (compara valor e tipo), além do operador != para verificar diferença. Foram apresentados os operadores lógicos && (and), || (or) e ! (not), destacando que o && exige que todas as condições sejam verdadeiras, enquanto o || necessita que apenas uma seja verdadeira. Também foi trabalhado o operador de módulo % para identificar números pares e ímpares, verificando se o resto da divisão por 2 é igual a zero com o uso do ===. A aula consolidou o entendimento de como expressões lógicas retornam valores booleanos (true ou false) e como esses resultados são utilizados na construção de regras e validações dentro dos programas.

1. Tipo Boolean
O tipo boolean representa apenas dois valores possíveis:
true
false


Ele é utilizado para representar condições lógicas, validações e resultados de comparações.
Exemplo:
const matriculado = true
const terminouOCurso = false
console.log(matriculado)
console.log(terminouOCurso)


2. Operadores de Comparação
Os operadores de comparação retornam sempre um valor booleano (true ou false).

2.1 Comparação com ==
O operador == compara apenas os valores, ignorando o tipo do dado.
Exemplo:
const number = 10
const number2 = '10'
const comparacao = number == number2
console.log(comparacao)

Explicação:
number é do tipo number.


number2 é do tipo string.


O == compara apenas os valores.


Como ambos representam o valor 10, o resultado será true.



2.2 Comparação com ===
O operador === compara valor e tipo.
Exemplo:



const number3 = 10
const number4 = '10'

const comparacao2 = number3 === number4
console.log(comparacao2)

Explicação:
O valor é igual.


O tipo é diferente (number e string).


Resultado: false.


Boa prática: utilizar sempre === para evitar conversões implícitas de tipo.

2.3 Operador Diferente (!=)
O operador != verifica se dois valores são diferentes.
Exemplo:
const numeroTeste = 10
const numeroTeste2 = 20
const operadorNot = numeroTeste != numeroTeste2
console.log(operadorNot)

Se os valores forem diferentes, retorna true.

3. Operadores Lógicos
Operadores lógicos permitem combinar múltiplas condições.
Principais operadores:
&& (and)


|| (or)


! (not)



3.1 Operador AND (&&)
Retorna true apenas se todas as condições forem verdadeiras.
Exemplo:
const number10 = 10
const number20 = 20
const number30 = 30
const operadorAnd = number10 < number20 && number20 < number30
console.log(operadorAnd)

Explicação:
10 < 20 → true


20 < 30 → true


Ambas são verdadeiras → resultado final: true


Se uma das comparações fosse falsa, o resultado seria false.

3.2 Operador OR (||)
Retorna true se pelo menos uma das condições for verdadeira.
Exemplo correto de uso:
const operadorOr = number10 > number20 || number20 < number30
console.log(operadorOr)

Explicação:
10 > 20 → false


20 < 30 → true


Como uma condição é verdadeira → resultado final: true



3.3 Operador NOT (!)
O operador ! inverte o valor booleano.
Exemplo:
const ativo = true

console.log(!ativo)
Resultado:
true se torna false



4. Operador de Módulo (%)
O operador % retorna o resto da divisão.
É muito utilizado para verificar se um número é par ou ímpar.
Exemplo:
const numeroPar = 10 % 2
const numeroImpar = 9 % 2
console.log(numeroPar)
console.log(numeroImpar)

Explicação:
10 % 2 = 0


9 % 2 = 1



5. Verificando se um Número é Par
Se o resto da divisão por 2 for igual a 0, o número é par.
Exemplo:
const numeroPar = 10 % 2
const par = numeroPar === 0
console.log(par)

Resultado: true

6. Verificando se um Número é Ímpar
Se o resto da divisão for diferente de 0, o número é ímpar.
Exemplo:
const numeroImpar = 9 % 2
const impar = numeroImpar === 0
console.log(impar)

Resultado: false

Nesse caso:

numeroImpar === 0 retorna false
Logo, sabemos que o número é ímpar

7. Consolidação dos Conceitos

== compara apenas valores.
=== compara valor e tipo (boa prática).
!= verifica se valores são diferentes.
&& exige que todas as condições sejam verdadeiras.
|| exige que ao menos uma condição seja verdadeira.
! inverte o valor lógico.
% retorna o resto da divisão.
Comparações sempre retornam boolean (true ou false).
Operadores lógicos são fundamentais para validações e estruturas condicionais.

Estruturas Condicionais em JavaScript (if e else)


1. Recapitulação: Operadores Lógicos

Antes de iniciar condicionais, foi feita a revisão de operadores lógicos, pois eles são a base para tomada de decisão em programas.

Exercício 1 – Verificando aprovação

const nota1 = 7
const nota2 = 9

const aprovado = nota1 >= 6 && nota2 >= 6

console.log("Exercício 1")
console.log("O aluno está aprovado?", aprovado)

Explicação detalhada

nota1 >= 6 verifica se a primeira nota é maior ou igual a 6.
nota2 >= 6 verifica se a segunda nota é maior ou igual a 6.
O operador && exige que as duas condições sejam verdadeiras.
Se ambas forem verdadeiras, aprovado será true.
Caso uma delas seja falsa, aprovado será false.
Esse é um exemplo clássico de regra de negócio: para ser aprovado, o aluno precisa atingir a nota mínima em ambas as avaliações.


Exercício 2 – Decisão com operador OR

Introdução a Objetos em JavaScript

Após a resolução do exercício, foi iniciado um novo conteúdo: Objetos em JavaScript.

Um objeto é uma estrutura utilizada para armazenar várias informações relacionadas entre si.

Essas informações são organizadas em pares de chave e valor, chamados de propriedades.

Cada propriedade possui:

um nome (chave)
um valor associado
Diferente dos arrays, que utilizam índices numéricos, os objetos são acessados utilizando o nome da propriedade.

Os objetos são muito úteis para representar informações mais complexas, como por exemplo:

uma pessoa
um carro
um produto
uma obra de arte
Além disso, um objeto pode armazenar diferentes tipos de dados, como textos, números ou até outros objetos.

Dentro dessas chaves colocamos as propriedades.

Exemplo — Objeto carro

let carro = {
  fabricante: "Ford",
  modelo: "Ka",
  ano: 2009
}

Neste objeto temos três propriedades:

Propriedade

Valor

fabricante

Ford

modelo

Ka

ano

2009


Exemplo — Objeto pessoa

let pessoa = {
  nome: "Fulano",
  idade: 18,
  cidadeDeNascimento: "Ponta Grossa, PR"
}

Este objeto representa uma pessoa e armazena três informações relacionadas a ela.


Acessando propriedades de um objeto

Para acessar uma informação dentro de um objeto utilizamos a notação de ponto.

console.log(carro.fabricante)

Nesse exemplo:

carro é o objeto
fabricante é a propriedade acessada
Saída no console:

Ford


Exercício — Criando um objeto de pintura

Durante a aula foi criado um objeto representando uma obra de arte.

let pintura = {
  nomePintura: "A Noite Estrelada",
  nomeArtista: "Van Gogh",
  ano: 1889,
  significado: "a vista da janela de um Hospício Francês"
}

Cada propriedade armazena uma informação relacionada à pintura.


Exibindo as informações do objeto

Depois foi utilizada template string para imprimir uma frase com os dados do objeto.

console.log(`A minha pintura favorita é ${pintura.nomePintura} de ${pintura.nomeArtista}. Feita em ${pintura.ano}, representa ${pintura.significado}!`)

Saída esperada:

A minha pintura favorita é A Noite Estrelada de Van Gogh. Feita em 1889, representa a vista da janela de um Hospício Francês!
